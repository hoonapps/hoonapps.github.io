Operating Model

블로그 운영 원칙

단순 요약보다 직접 실행한 증거, 아키텍처 선택의 이유, 운영에서 터질 수 있는 실패 모드를 우선합니다. 좋은 글은 지식보다 판단력을 남겨야 합니다.

  1. 기술은 직접 실행한다. README 요약이 아니라 설치, 예제, 한계, 대체재를 확인합니다.
  2. 백엔드는 내부 동작까지 간다. API 사용법에서 멈추지 않고 DB, 네트워크, 런타임, 장애 지점을 봅니다.
  3. 프로젝트는 결정 기록을 남긴다. 무엇을 만들었는지보다 왜 그렇게 만들었는지를 추적합니다.
